    Socket.IO Client

    WolfClient handles all Wolfringo-high level connectivity concerns. But under the hood, is uses a different service - ISocketClient. This service is designed to handle all low-level Socket.IO (transport protocol used by WOLF Protocol v3) that WolfClient requires.

    ISocketClient is responsible for maintaining WebSocket connection to with WOLF Server and responding to ping packets, as well as sending and receiving carefully serialized/deserialized low-level SocketMessages that are compatible with Socket.IO protocol.

    You can check the default implementation on GitHub: SocketClient.cs.

    Custom Socket Client

    The default implementation of Socket Client is designed to work with Wolfringo specifically, so it might not support all Socket.IO protocol features. If you need to extend or replace the socket client code, you can create a new class that implements ISocketClient interface. This interface specifies a few members:

    Properties:

    • IsConnected - indicates whether the client is currently connected.

    Events:

    • Connected - when the client establishes a connection with the server.
    • Disconnected - when the client loses the connection with the server.
    • MessageReceived - when the client receives a message from the server.
    • MessageSent - when the client successfuly sends a message to the server.
    • ErrorRaised - when there has been an exception in connection handling loop.

    Methods:

    • ConnectAsync(Uri, CancellationToken) - connects to the server on a given URI.
    • DisconnectAsync(CancellationToken) - disconnects from the server.
    • SendAsync(JToken, IEnumerable<Byte[]>, CancellationToken) - sends a message to the server.

    Once your custom class is finished, you need to register it with WolfClientBuilder as explained in Introduction.

    Warning

    Creating a custom socket client is a delicate task. ISocketClient is the fundamental building block for entire Wolfringo library, and WebSocket networking code and connection loop is very error-prone.
    For this reason, it's recommended to NOT create custom Socket Client and use Wolfringo's implementation, unless customization is really necessary.
