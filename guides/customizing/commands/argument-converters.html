    Customizing Argument Conversion

    When your command method executes, Commands System needs to convert string argument into whatever type you set in your method for that argument. It does it by using a set of IArgumentConverters.

    Creating a new Argument Converter

    Creating an Argument Converter is very easy - simply create a new class and implement IArgumentConverter interface. IArgumentConverter requires 2 methods:

    • CanConvert - determines if this converter can convert to given method parameter at all.
    • Convert - converts the string argument into fitting method parameter.

    Here is an Int32Converter implementation for example:

    public class Int32Converter : IArgumentConverter
{
    public bool CanConvert(ParameterInfo parameter)
        => typeof(Int32) == parameter.ParameterType;

    public object Convert(ParameterInfo parameter, string arg)
        => System.Convert.ToInt32(arg, CultureInfo.InvariantCulture);
}

    Registering Custom Argument Converter

    In order for Wolfringo Commands System to know about your converter, you need to register it with IArgumentConverterProvider. Default ArgumentConverterProvider implementation in Wolfringo uses ArgumentConverterProviderOptions with a dictionary map and explicit enum converter, so for most use cases, you don't even need to create a custom one.

    • Without Wolfringo.Hosting (Normal Bot)
    • With Wolfringo.Hosting (.NET Generic Host/ASP.NET Core)

    Note: instructions skip logger setup. See Logging guide for details.

    1. Manually create an instance of ArgumentConverterProviderOptions.
    2. Add your converter to Converters dictionary, optionally change EnumConverter.
    3. Use WithDefaultArgumentsConverterProvider method of CommandsServiceBuilder and pass your options as argument.
    ArgumentConverterProviderOptions options = new ArgumentConverterProviderOptions();
options.Converters[typeof(MySpecialType)] = new MySpecialTypeConverter();
_client = new WolfClientBuilder()
    .WithCommands(commands => 
    {
        commands.WithDefaultArgumentsConverterProvider(options);
    })
    .Build();
    1. Configure ArgumentConverterProviderOptions.
    2. Add your converter to Converters dictionary, optionally change EnumConverter.
    services.Configure<ArgumentConverterProviderOptions>(options =>
{
    options.Converters[typeof(MySpecialType)] = new MySpecialTypeConverter();
});

    Custom ConverterProvider

    The default ArgumentConverterProvider implementation will first check Converters for specific parameter type. If it doesn't exist, it'll check if the type is an enum, and if it is, it'll return EnumConverter.

    If your requirements are more complex, you can write your own class that implements IArgumentConverterProvider and register it with CommandsServiceBuilder as explained in Introduction.

    You can view default implementation on GitHub for reference.
