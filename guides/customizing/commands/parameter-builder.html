    Custom Parameter Builder

    IParameterBuilder is responsible for actually injecting values of Commands' method arguments.
    The default ParameterBuilder implementation works as follows:

    1. Check if it's a special argument provided by Command Instance. This is mainly used by RegexCommandInstance to provide regex Match to the command method.
    2. Provide ICommandContext, or any type contained in the context.
    3. Provide calculated ICommandOptions for currently executed command context.
    4. Provide ICommandInstance that executes the method.
    5. Provide Command Service's @System.Threading.CancellationToken.
    6. Provide Command's Arguments raw text if argument is a string and has [ArgumentsText] attribute.
    7. Resolve services from @System.IServiceProvider. This includes generic loggers.
    8. Attempt to use any of registered IArgumentConverters.

    If no match is found through all these steps, the Builder will report a failure and command execution will fail.

    Custom Parameter Builder

    If the flow above doesn't meet your needs, you can create a custom class implementing IParameterBuilder interface. This interface has only one method - BuildParamsAsync, which takes following parameters:

    • IEnumerable<ParameterInfo> parameters - enumerable of each parameter in the command method being executed, in order.
    • ParameterBuilderValues values - set of values provided by ICommandInstance executing the method. This includes @System.IServiceProvider, ICommandContext, the parsed arguments of the command (as string array), and some more things that might be required.
    • CancellationToken cancellationToken - cancellation token that can cancel the building. Note: this is only for builder, the CancellationToken to be used as a method parameter exists in ParameterBuilderValues.

    This method returns a ParameterBuildingResult. The most important property is Values - array of actual values to inject into the command method, in the same order as IEnumerable<ParameterInfo> parameters provided.

    If you wish to see default ParameterBuilder implementation for reference, it is available on GitHub.

    Once your custom Parameter Builder is ready, you need to register it with CommandsServiceBuilder as explained in Introduction.
