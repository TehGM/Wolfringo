    Customizing Arguments Parser

    IArgumentsParser is used by Wolfringo Commands System whenever Standard commands are involved (Regex commands do not use the Arguments Parser). It is what separates arguments by space, and groups them together if put in quotes like "foo bar" etc.

    Like many other parts of Wolfringo Commands System, IArgumentsParser is resolved by CommandsService from @System.IServiceProvider - and therefore you can fully overwrite the way Worlfringo Standard commands parse arguments. Default ArgumentsParser implementation also allows changing its options, so you can tune it to your needs even without writing your custom one!

    Changing Options of Default Parser

    Default ArgumentsParser implementation takes ArgumentsParserOptions as its constructor argument. You can use that to change the behaviour of the default Arguments Parser easily before Commands Service starts.

    Options you can change include:

    • BaseMarker - the character that is used to split arguments in most scenarios. Defaults to space (so arguments are per-word).
    • BlockMarkers - dictionary of opening and closing characters that define a block. Defaults to 2 spaces, [ and ], ( and ), and " and ".
    • InitialBlockSizeAllocation - performance option. Changes the default memory allocation per argument parsed before it grows. Defaults to 8, which should be enough for most commands arguments.
    • Without Wolfringo.Hosting (Normal Bot)
    • With Wolfringo.Hosting (.NET Generic Host/ASP.NET Core)

    Note: instructions skip logger setup. See Logging guide for details.

    1. Manually create an instance of ArgumentsParserOptions.
    2. Customize the options however you need.
    3. Use WithDefaultArgumentsParser method of CommandsServiceBuilder and pass your options as argument.
    ArgumentsParserOptions options = new ArgumentsParserOptions();
options.BaseMarker = '-';
_client = new WolfClientBuilder()
    .WithCommands(commands => 
    {
        commands.WithDefaultArgumentsParser(options);
    })
    .Build();
    1. Configure ArgumentsParserOptions.
    2. Customize the options however you need.
    services.Configure<ArgumentsParserOptions>(options =>
{
    options.BaseMarker = '-';
});
    Tip

    You can also configure these options using other means of .NET Generic Host configuration, such as JSON file.

    Custom Parser

    While the default ArgumentsParser implementation allows for some customizability, you can get full control by writing your own implementation. To do so, simply create a class implementing IArgumentsParser interface. Once your class is complete, you can register it with CommandsServiceBuilder as explained in Introduction.

    You can view default implementation on GitHub for reference.
